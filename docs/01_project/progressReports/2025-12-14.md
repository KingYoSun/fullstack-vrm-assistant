# 2025年12月14日 進捗レポート

## 概要
- デフォルト待機モーションとして `frontend/public/idle_loop.vrma` を自動再生し、Motion/VRMA 再生後に待機へ復帰する挙動を追加。
- `idle_loop.vrma` の取得元が ChatVRM である旨を README に明記。

## 成果
- `frontend/src/components/avatar/AvatarCanvas.tsx`: VRM 読み込み後に待機 VRMA をロードして LoopRepeat 再生し、`AnimationMixer` finished で待機へ復帰。
- `frontend/src/utils/vrmaLoader.ts`: `loadVrmaClip` を URL 引数のみに整理し、VRMA ロード周りの型を補強。
- `frontend/src/store/appStore.ts`: 重複していた `setVrmaUrl`/`playVrma` のエクスポートを整理。
- `README.md`: ChatVRM 由来の待機モーションである旨を追記。
- 検証: `pnpm -C frontend exec tsc -b`, `pnpm -C frontend lint`（警告のみ）, `pnpm -C frontend exec vite build --configLoader native`

## 課題
- `vite build` のデフォルト configLoader（bundle）は `node_modules/.vite-temp` への書き込み権限に依存し、環境によっては `--configLoader native` が必要。
